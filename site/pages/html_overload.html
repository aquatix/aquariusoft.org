overload RSS reader

I'm currently developing a webbased, Open Source [GPLed] news feed reader, which you can use on this server, or download and install on your own webserver, needing only PHP and a MySQL database.

overload is based on the magpierss parser [currently version 0.71.1] and is written in pure PHP using only this parser and a MySQL database.

Relatively stable snapshots will be used to power the reader on this server [log in for a demonstration with the user/password combo demo/demo] and the source will become available when it's usable for third parties, but expect a tarball soon.

You can also use overload through secured http [https, SSL], which is secured by a self-signed security certificate, which you'll get a warning about. This is OK in this case, as you're not using it to identify us, but for securing your information over the internet. Be wary of those warnings in other cases, though!

Roadmap

Version 0.1 [due June] -- publicly usable version with basic but powerful featureset

  • underlying engine
  • subscribing as new user
  • add feeds
  • manage [modify, delete] feeds
  • manage [modify, delete] groups
  • flag interesting postings [so you can find them easily later on]
  • pda access through usercode
  • XUL rich client
  • related items [look for related postings based on shared links]

Version 0.2 [July, August?] -- extended features and user interface

  • foldable groups
  • count of unread items per group [maybe for folded groups only]
  • spreaded downloading of feeds, not all in one large sequence once an hour
  • exporting OPML files
  • importing OPML files
  • user defined filters [based on keywords, date etc. Please contact me for more ideas!]
  • tagging support for postings
  • tagging support for feeds [replacing groups?]
  • user choosable stylesheets

Future versions

  • favicons for the feeds [through generated stylesheet file containing the uri's of the pictures]
  • resyndication of feeds [Planet-like behaviour]
  • firefox plugin for fast access of your feeds [and adding of new feeds]
  • XML based API for using with standalone tools
  • other ideas from this interesting post [read the comments too]
  • "e-mail this" feature
  • sort of scripting support? Figure out a good way [maybe query-like, and integrate it with the user defined filters]
